JSON format

JSON (Javascript Object Notation) is a standard text-based format. It is represented in key value pairs. The key is always a string, but the value can be a string, number, boolean, array. Its commonly used for transmitting data in webapplications and is often used restAPIs. Javascript has a global JSON object that has methods available to convert between JSON and javascripts objects. Which makes it easy to work with in javascript.

        
{
    "name": "Cam",
    "age": 35,
    "Hobbies": ["music, fitness", "programming", "travels"]
}

Fetch

When working with a seperat backend and frontend you often work with two different programming language, like we did in 3 semester. The way we connected our frontend with our backend was to fetch the data with fetch()

fetch() is s built in javascript function. It takes two arguments in form of an URL it fetches data from, and options if they need to be defined. Options is an is optional object where additional information can be set, like method(if its a POST, PATCH, PUT, DELETE request), headers that includes credentials, body that contains data to be sent with the request and some more optional properties.



Theres two ways of making a fetch method, which one chosen is more a matter of preference.


        
//We make the function async so its continues even though fetch havent returned the promise object.
//But since we use the resonse we set it to be await so the code doesnt continues before it receives the response.
//The response we get from the url is in JSON and we then parse to a javascript object so we can work with it.
async function fetchSomething(){
    const url= 'www.localhost:8080/users'
    const response = await fetch(url)
    const data = response.json()
}


Another way to fetch;

        
//.then() is a method to handles promises when the response is available.
fetch('http:wwww.localhost:8080/users').then(response => resonse.json()).then(result => console.log(result))
