Modules and express

In nodejs we work with modules which is a unit of code that can be reused in other part of the program. In nodejs, modules are typically js files. Which is what will be focused on here.

Express is a webapplikation framework(dependency/library) for Nodejs. Express provides a set of features that simplify the process of building web applications. Its contains different features like; routing, template engines, static file serving, errorhandling etc.

Serving static files; refers to the process of serving static files such as css, html, javascript, images etc. to the clients web browser. This means that the client has access in the browser in the given directory.

app.use(express.static('public'))

express.static takes the name of the directory containing the static files, as an argument. When a client request a static file, the middleware function checks if the file exists in the public folder. This basicly means that the client can acces any static file stored in the public folder.

This is a better solution than handling files manually like done below: 

            
                app.get('/', (req, res) => {
                  // Read the contents of the file
                  fs.readFile('public/index.html', (err, data) => {
                    if (err) {
                      // Handle the error
                      res.status(500).send('Error reading file');
                    } else {
                      // Send the file contents as the response
                      res.send(data);
                    }
                  });
                });
                
                // Start the server
                app.listen(8080, () => {
                  console.log('Server listening on port 8080');
                });
